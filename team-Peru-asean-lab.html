1.3.3 Observations and surprises

Year runs backwards. The data opens at 2021 and closes at 2018 — most people would expect chronological order (oldest first), but it’s newest-first.

Countries are alphabetical within each group. Brunei → Cambodia → Indonesia → Lao PDR → Malaysia → Myanmar in the head; Malaysia → Myanmar → Philippines → Singapore → Thailand → Viet Nam in the tail.

“Viet Nam” is two words. It appears as “Viet Nam” rather than the more commonly typed “Vietnam”, which could cause a mismatch if you ever join this dataset against another source.

2.3.3 Summary of missing production values and total production

The code confirms that the production column contains missing values because any(is.na(meat$production)) returns TRUE. We then used the logical mask is.na(meat$production) to filter the dataset and display only the rows where production is missing. The missing rows are shown using only the country, animal, and year columns so that we can clearly identify where the missing production values occur.

To cross-check the number of missing rows, we calculated the expected decrease in rows by summing the number of NA values in the production column. We then created a new dataset called meat_complete that excludes rows with missing production values. By comparing the actual decrease in rows with the expected decrease, we confirmed that they match. This validates our identification of the missing values. The row count decreases from 160 to 156, confirming that there are 4 missing production values in the dataset.

For the total production calculation, sum(meat$production) returns NA because the production column still contains missing values. R does not know how to add an unknown value, so the whole result becomes NA. When we use sum(meat$production, na.rm = TRUE), R ignores the 4 missing values and adds up the remaining 156 known production values. This returns 83,937.52, which represents the total ASEAN meat production from the non-missing records.

2.3.4 Assumption made by using na.rm = TRUE

Using na.rm = TRUE assumes that it is acceptable to ignore the missing production values instead of estimating or replacing them. This means the missing rows do not contribute to the total production calculation.

This assumption is acceptable for a simple exploratory calculation because only 4 out of 160 production values are missing. However, it is not perfect because those missing rows still represent real country-animal-year records. If the missing production values were large, ignoring them would underestimate the true total ASEAN meat production.

4.3 Should meat$animal_factor Be An Ordered Factor?

The output of levels(meat$animal_factor) shows that the animal factor has four levels: beef and buffalo, mutton and goat, pig, and poultry. The alphabetical check returns TRUE, which means the levels are arranged alphabetically.

meat$animal_factor should not be an ordered factor because the animal categories do not have a natural ranking. For example, beef and buffalo, mutton and goat, pig, and poultry are different categories of meat, but one is not naturally higher or lower than another. An ordered factor would be more suitable for categories such as low, medium, and high, where the order has meaning. Therefore, it is appropriate for meat$animal_factor to be an unordered factor.

This indicates that the collapse was successful, and the new color factor correctly categorizes the meats into red and white groups.
